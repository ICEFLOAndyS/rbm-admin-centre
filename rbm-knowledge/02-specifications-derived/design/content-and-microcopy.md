> **Classification:** Reference Implementation (Recommended)  
> **Scope:** RBM Design System  
> **Enforcement:** Advisory  
> **Deviation:** Permitted if normative standards are met
## 1. Tone and style
- Clear and unambiguous
- Use operational language (actions, outcomes)
- Avoid jargon unless it is standard in ops/resilience contexts

## 2. Labels
- Action verbs: Start, Pause, Resume, Approve, Reject, Escalate
- Avoid generic labels: Submit, OK, Continue (unless contextualised)

## 3. Evidence prompts
Where evidence is required, prompt clearly:
- What was done?
- What was observed?
- Who approved?
- What evidence is attached?

## 4. Notification text
- Include object reference (Event/Runbook ID)
- Include current status and next required action
- Include link back to the record
